Hudson Valley AI / Deploy-your-own runbook

How a deploy-your-own engagement actually works.

Phase-by-phase walkthrough of what we do, what you do, how long each step takes, and what infrastructure you'll be paying for on an ongoing basis. The honest version. Nothing handwaved.

If you've decided to deploy skateOS or Branch Manager on your own infrastructure (instead of staying on our hosted instance), here's what the engagement looks like from signed engagement letter to live system. About 3-5 weeks end-to-end depending on data-migration complexity.

Before we start: You'll need to be available for ~30 minutes on Day 1 for account setup, ~2 hours total spread across the first week for decisions + branding choices, and ~3-4 hours of staff training in the final week. Nothing else is required of you beyond reviewing what we ship.
1

Cloud-account provisioning

Day 1-2 · 30-min call + async

Step zero of ownership: the cloud accounts have to be in your name, not ours. We walk you through creating each account on a screen-share call, you receive the welcome emails, you set the master passwords. We never have ownership of any of these — only scoped access we'll revoke at handoff.

What we do

  • Walk you through Supabase signup live on a screen-share
  • Help you pick the right region (US-East for Hudson Valley)
  • Configure Cloudflare DNS for your domain
  • Set up Resend sending domain verification
  • Configure Twilio number provisioning if SMS in scope

What you provide

  • A credit card (paying these vendors directly — not us)
  • Domain name (we'll help you register if needed)
  • Business email for account ownership
  • ~30 minutes on a video call, Day 1

Accounts you'll create

  • Supabase — Postgres database + auth + storage + edge functions. Free tier covers low-volume; Pro ($25/mo) kicks in around 200 daily-active users.
  • Cloudflare — DNS + static-asset hosting via Workers. Free tier for both at typical SMB volume.
  • GitHub — source code repository. Free tier fine for private repos.
  • Resend — transactional email. Free tier = 3,000 emails/month; $20/mo for 50k.
  • Twilio — SMS (optional, recommended). Pay-per-message at ~$0.0079 each.
  • Helcim (skateOS) or Stripe Connect (Branch Manager) — payment processing direct merchant account. No SaaS markup, interchange-plus pricing.
2

Code + database installation

Day 3-7 · async on our side

We push the codebase into your GitHub repo and apply the database schema to your Supabase project. You watch happen via shared screens or recordings if you want; nothing requires your active participation here.

What we do

  • Push the canonical skateOS or Branch Manager codebase to your GitHub
  • Apply the migration sequence (60+ migrations, idempotent, ordered)
  • Deploy the static admin SPA to your Cloudflare Workers
  • Deploy ~17 Edge Functions to your Supabase
  • Configure Vault secrets (Helcim API token, Resend API key, etc.)
  • Set up pg_cron schedules (lesson reminders, daily digest, backups)
  • Configure webhook endpoints (Helcim/Stripe, Smartwaiver)

What you provide

  • Approval to push code into your repo (one click)
  • API keys you've received from Resend/Twilio/etc. (paste them in our shared secrets doc)

What's installed

  • Database — ~60 tables across CRM, sales, lessons, products, equipment, multi-tenant infrastructure, audit log
  • Admin SPA — web app at app.yourdomain.com for staff to operate the business
  • Edge Functions — payments (Helcim/Stripe), email (Resend), SMS (Twilio), waiver lookup (Smartwaiver), daily-digest, backup, etc.
  • pg_cron schedules — hourly lesson reminders, daily renewal reminders, daily digest at 8am ET, weekly backup, etc.
  • RLS policies — Row-Level Security on every business table, so your data is isolated at the database layer (not just the app layer)
3

Branding + business configuration

Day 8-14 · ~2 hr of your time spread out

Time for the system to actually look and behave like your business, not 2nd Nature Park or 2nd Nature Tree. We work through this together on a series of short async exchanges + one or two short calls.

What we do

  • Apply your colors, logo, business name everywhere
  • Configure operating hours, tax rates, sales tax registration
  • Set up your product/service catalog
  • Configure staff accounts with roles + permissions
  • Set up customer-portal magic-link auth
  • Configure payment processor webhooks + test flow
  • Wire up email/SMS templates with your branding
  • Set business-specific feature flags (mobile bus, equipment loaners, etc.)

What you provide

  • Logo file (SVG ideal, PNG fine)
  • Brand colors (or we'll suggest from your existing site)
  • Business address, phone, EIN for receipt footers
  • Tax rate + sales-tax registration number
  • Operating hours, day-of-week schedule
  • Product/service catalog (we'll help format)
  • Staff names + email addresses + roles
  • ~2 hours total spread across 5-7 days, mostly async
4

Data migration from your current system

Day 15-21 · varies by source system

The "leaving your current SaaS" part. Most painful when source data is messy (it usually is), but doable. We've migrated from Square, Mindbody, Jobber, ServiceTitan, Smartwaiver, and from paper / spreadsheet operations.

What we do

  • Pull export from your existing system (CSV, API, scraping if needed)
  • Map fields from source schema to skateOS/BM schema
  • Transform + clean the data (de-dupe customers, fix bad dates, etc.)
  • Load into your Supabase, validate row counts
  • Reconciliation pass: sample 10-20 records by hand against source
  • Plan migration cutover window

What you provide

  • Credentials to export from your existing system, OR exported CSVs/JSON
  • Approval on the migration cutover window (typically a low-traffic day)
  • Review of the reconciliation sample to confirm data integrity

Source systems we know

  • Square POS / Square for Restaurants — CSV export, well-documented schema
  • Mindbody — paid data export, painful but doable
  • Jobber — CSV per data type, straightforward
  • ServiceTitan — API export available with their support
  • Smartwaiver — direct integration in skateOS already (no migration needed; just connect)
  • Stripe / Shopify — clean CSV exports
  • Paper / spreadsheets — case by case; we'll quote
5

Staff training + parallel run

Day 22-28 · ~3-4 hr of staff time

Where most software rollouts fail isn't the software — it's that the staff don't know how to use it. We do live training, record it, and run a 1-2 week parallel period where both systems are live.

What we do

  • 2-hour live training session for all staff (recorded for future hires)
  • Role-specific cheat sheets (front desk, owner, instructor, etc.)
  • Daily check-ins during parallel-run week
  • Fix any "I tried to do X and it didn't work" issues same-day
  • Document any final tweaks needed before full cutover

What you provide

  • Staff availability for the 2-hour training
  • Willingness to run parallel for 1-2 weeks (extra register entry, but lowers cutover risk)
  • Honest feedback on what's confusing or broken
6

Go-live + handoff

Day 29-35 · we step back, you take the keys

Cutover day. Old system gets decommissioned, new system handles all transactions, and we revoke our access — you officially own the stack.

What we do

  • Final cutover: flip DNS, point payment webhooks at new system
  • Watch for issues in the first 48 hours (we're on standby)
  • Cancel your legacy SaaS subscription (you confirm; we walk you through it)
  • Transfer all credentials, secrets, runbooks to your control
  • Revoke our access to your Supabase / GitHub / Cloudflare
  • Provide a written attestation of access removal
  • 30-day post-delivery support window for any issues

What you walk away with

  • Source code in your git repo, full history
  • Database in your Supabase project, accessible by you
  • Cloud accounts all in your name, paid directly by you
  • Runbooks for deploy, rollback, common incidents
  • Architecture doc — every system, every dependency
  • 30 days free post-delivery support
  • Optional maintenance retainer — $1-3k/yr for ongoing updates

Common questions

What if I want to change something after handoff?

Three options: (1) modify it yourself / hire any developer (it's your code now), (2) come back to us for a $3k Fix-it Sprint, (3) put us on retainer at $1-3k/yr for quarterly updates.

What if Hudson Valley AI disappears tomorrow?

You keep running. Source code is in your git, database in your Supabase, cloud accounts in your name. The only thing you lose is our future support — and any competent developer can pick up the codebase since it's standard Postgres + vanilla JS + Cloudflare Workers (no exotic frameworks, no proprietary anything).

What if I outgrow the platform and need to leave?

Export your data from Postgres anytime. It's standard SQL — every other system can ingest it. You'll lose the skateOS/BM-specific workflows, but no SaaS lock-in keeps you stuck.

Do you do this for free trials too?

No — trials run on our hosted instance for the 30-day evaluation period. The full deploy-your-own engagement happens after you decide to convert. Trial data exports to your new tenant on Day 30 of the trial if you choose deploy-your-own.
