The problem

Independent skateparks run their operations on a stack of single-purpose tools: Square for POS, Mindbody for memberships, Smartwaiver for liability, a Squarespace site for marketing, Stripe Payouts for payments, MailChimp for marketing emails, paper for incident reports. Nothing talks to each other. Customer data lives in five places. Reports are spreadsheet stitch-jobs.

2nd Nature Park, an independent skatepark in Peekskill NY, hired us to replace the entire stack with one accountable system.

"We were spending 8 hours a week reconciling data across tools that didn't talk. We needed one system, built for skateparks specifically, that we could actually trust." — 2nd Nature Park founders

What we built

A multi-tenant SaaS operating system covering the full skatepark workflow:

  • Front-desk check-in with QR member-card scan, voice search, real-time park-capacity tracking, waiver enforcement, age-gate logic for minors.
  • Point of sale with cart memo, discount codes, gift cards, loyalty redemption, split payments, tip allocation, cash-drawer reconciliation, end-of-shift tape printing.
  • Lessons & bookings with calendar, instructor scheduling, group attendance tracking, automated reminders + post-lesson follow-ups, waitlist promotion, online booking via Helcim.
  • Memberships with monthly / annual / punch-card / day-pass plan types, pause + auto-resume, customer self-serve cancel.
  • Customer self-serve portal — magic-link auth, edit profile, view orders, cancel lessons, manage memberships, print member card, view loyalty points, hours-skated stat, achievement badges.
  • Online retail shop with public catalog, cart, checkout, order tracking, automated payment + shipping confirmation emails.
  • Equipment / loaner gear tracking with check-out / check-in / damage tracking / overdue reminders.
  • Mobile bus shop for off-site events with separate inventory tracking + run reconciliation.
  • Affiliate / team rider program with commission tracking, public profiles, share-codes, payout management.
  • Incident logging with structured fields for liability + insurance compliance.
  • Marketing — campaign builder with tag-based segmentation, A/B test split, CAN-SPAM-compliant unsubscribe, Twilio SMS opt-out compliance.
  • Reports + BI — 30+ analytical cards covering revenue, retention, cohort, LTV, churn, lesson cancellation reasons, instructor utilization, per-product margin.
  • iPad-native mobile app (Expo / React Native) for front-desk + bus-shop use.

Security choices & why

Every architectural decision was made with security implications front of mind. The full security & compliance posture is published; here are the highlights specific to skateOS:

  • Postgres RLS for multi-tenancy — even if application-layer auth fails, the database refuses cross-tenant reads. Cross-tenant data exposure is technically impossible without compromising the database itself.
  • Card data never on our infrastructure — Helcim's hosted iframe captures cards, returns a tokenized transaction reference. We store only the reference, never the PAN. PCI scope: SAQ-A or lower.
  • Customer self-serve portal isolation — RLS policies restrict authenticated customers to read-only access on their own record + sales + lessons + loyalty. No cross-customer leakage even on application bugs.
  • Webhook signature verification on Helcim and Smartwaiver inbound webhooks. Replay-protection via idempotency keys.
  • Audit log via Postgres trigger — every INSERT/UPDATE/DELETE on business-critical tables captured with actor identity and old/new values. Full forensic trail for "who changed what when" investigations.
  • CAN-SPAM + TCPA compliance built in — public unsubscribe RPC, tag-based SMS opt-out, opt-out gates on every commercial-email send path.
  • Encrypted at rest — Supabase Postgres + S3-equivalent backup storage, all AES-256.
  • HTTPS / TLS 1.2+ via Cloudflare on every public surface. HSTS headers. No HTTP fallback.
  • Secret management — all API keys / signing tokens stored in Supabase secrets, never in client code or git.
  • Honeypot anti-spam on public forms (intake, lesson booking).

How we shipped this fast

61 database migrations and 100+ features in a few months. The velocity came from three things:

  • AI as a force multiplier. Every feature was specced, written, and reviewed with AI-assisted tooling. This is not "AI generates the code unsupervised" — it's "principal engineer + AI tooling = 5× output." Quality is enforced by the principal, not by the AI.
  • Daily ship cadence. Features ship the day they're discussed. No two-week sprints, no waiting for the next planning meeting. Bugs and feedback land within the same business day.
  • Minimal framework choices. Vanilla JS for the frontend, Postgres for the database, Cloudflare Workers for static assets, Supabase Edge Functions for server logic. Boring, fast, observable, scales without rearchitecting.

Outcomes

  • Replaced Square POS, Mindbody, and 4 other tools with one accountable system.
  • Reconciliation time cut from ~8 hours/week to under 30 minutes.
  • Customer-self-serve portal deflected an estimated ~40% of routine front-desk calls (membership balance checks, lesson cancellations, receipt requests).
  • Zero security incidents over the build period.
  • Now expanding as white-label SaaS for additional skatepark deployments.
